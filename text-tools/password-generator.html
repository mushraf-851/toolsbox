How does this password generator work?

Every password is generated using crypto.getRandomValues() — the Web Crypto API built into your browser. This is cryptographically secure randomness, not Math.random(). No password ever reaches any server.

🔒 Cryptographically secure

Uses window.crypto.getRandomValues — the same API used by password managers and security software. True randomness, not pseudo-random.

📏 Password entropy

Entropy (bits) = length × log₂(pool). A 16-char password with all types has ~105 bits of entropy — would take billions of years to brute-force.

🚫 Nothing stored

Your passwords are never sent anywhere. Everything happens in your browser tab. Close the tab and the passwords are gone.

🔑 Use a manager

Generate unique passwords here, then store them in Bitwarden, 1Password, or your browser's built-in manager. Never reuse passwords.

Frequently asked questions

Is this password generator safe?
Yes. All passwords are generated in your browser using crypto.getRandomValues — the Web Crypto API. Nothing is sent to any server. Your passwords are completely private.
What makes a password strong?
At least 16 characters, mixing uppercase, lowercase, numbers and symbols. Avoid dictionary words or personal info. Our strength meter shows entropy in bits — aim for 80+ bits.
Why exclude ambiguous characters?
Characters like 0 (zero), O (capital-o), l (lowercase-L), and 1 (one) look similar in many fonts. Excluding them makes passwords easier to type manually without mistakes.
How long should my password be?
16+ characters for most accounts. 24+ for email, banking, and password managers. Longer is always better — the generator supports up to 128 characters.
What is entropy in passwords?
Entropy measures how unpredictable a password is. Calculated as: length × log₂(character pool size). 80+ bits = strong, 100+ bits = very strong, 128+ bits = overkill-proof.
